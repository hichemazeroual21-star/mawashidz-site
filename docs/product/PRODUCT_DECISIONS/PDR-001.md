# PDR-001: Smart Workspace replaces traditional Dashboard

| | |
|--|--|
| **Status** | Accepted |
| **Date** | 2026-07-23 |
| **Owner** | Founder / Product |

## Title

Smart Workspace replaces traditional Dashboard

## Reason

Increase **daily engagement** and **operational efficiency**. Members should open MawashiDZ every day—not only when buying or selling. A passive dashboard that only displays metrics does not support sell, buy, manage, and act in one place.

## Alternatives considered

| Alternative | Why not chosen |
|-------------|----------------|
| **Classic dashboard** | Information-only; weak habit formation; does not unify tasks, listings, and messages |
| **Multiple role dashboards** | Fragmented UX; duplicate chrome; harder to add cross-cutting Hub and notifications |
| **Hybrid** (dashboard + scattered tools) | Grows into inconsistent navigation; every new feature asks for a new page |

## Decision

Adopt a **unified Smart Workspace** per role (and rank): five pillars (insights, operations, quick actions, statistics, notifications). The member profile is **not** an account settings page—it is the **daily operational home**.

Use the term **Smart Workspace** in product copy and new specs; avoid labeling member surfaces as “Dashboard.”

## Consequences

- **Design gate:** every new feature must answer *“Which workspace benefits from it?”* If none, reconsider the feature.
- Future services integrate as **workspace sections** and/or **Hub Cards**, not one-off account tabs.
- Legacy code names (e.g. `mdz-dashboards.mjs`) may remain until refactored; **new work** uses workspace language.
